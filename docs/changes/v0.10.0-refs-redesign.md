# v0.10.0: Refs Redesign — Unified @-Syntax, Grouped Refs, Category Validation

**Version**: v0.10.0
**Date**: 2026-05-21

---

## Overview

Major redesign of the reference system. This release shifts refs from a typed-entry array to a grouped path map, unifies reference syntax under a single `@` prefix, and introduces category-level document validation rules.

**Breaking change**: frontmatter `refs` format is not backward-compatible with v0.9.0. Run `opsv refs sync <file> --write` to migrate documents.

---

## Reference Syntax (unified `@` prefix)

| Form | Meaning |
|------|---------|
| `@id` | External asset |
| `@id:variant` | External + variant |
| `@:key` | In-document Design Reference |
| `@FRAME:shotId_first/last` | Frame ref (resolved at compile time, not in refs) |

## Frontmatter Refs (grouped structure)

```yaml
refs:
  image:
    "@hero":
      - opsv-queue/.../hero_1.png
    "@:angle_side":
      - ./refs/hero_side.png
    "@style:night":
      - opsv-queue/.../style_night.png
  video:
    "@swim_loop":
      - opsv-queue/.../swim.mp4
```

- Outer key: input_type (must be registered in `.opsv/input_types.yaml`)
- Inner key: canonical @-syntax key
- Value: non-empty array of paths

## Bidirectional Validation

prompt + visual_brief + visual_detailed scanned for @-tokens; must equal refs keys exactly.
Configurable per category in `.opsv/category_validate.yaml` via `refs_in_prompt_must_match_refs: true`.

## Removed

- `RefEntry`, `RefEntrySchema`, `TypedSectionRef`
- `parseRefs`, `parseTypedSections`, `resolveToInputs` (replaced by `bindRefs`)
- `[title](#id)` internal link syntax (replaced by `@:key`)
- `### <type>` subsections as ref parsers (now pure markdown organization)

## CLI Additions

```bash
# refs check / sync
opsv refs check <file>
opsv refs sync <file> --write

# validate enhancements
opsv validate --category shot
opsv validate --strict                # warnings → failures
opsv validate --skip-category-rules

# produce commands
opsv imagen --model X --prompt-mode index
opsv animate --model X --prompt-mode keep
opsv comfy --model X --prompt-mode name
opsv webapp --model X --prompt-mode keep
```

## Prompt Compile Modes

| Mode | Behavior |
|------|----------|
| `keep` (default) | prompt unchanged; `_opsv._refs_map` attached |
| `index` | `@hero` → `image1`, `@:angle_side` → `image2` |
| `name`  | `@hero` → `hero`, `@:angle_side` → `angle_side` |

Resolution order: CLI `--prompt-mode` > api_config `prompt_compile_mode` > default `keep`.

## Migration Guide

1. Update `frontmatter.refs` to grouped structure (run `opsv refs sync <file> --write`)
2. Replace `[label](#id)` internal links with `@:key` in prompt fields
3. Add `.opsv/input_types.yaml` (auto-created by `opsv init`)
4. Optional: add `videospec/_category_validate.yaml` to enforce per-category rules
5. Verify with `opsv validate --strict`
