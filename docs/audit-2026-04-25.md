## 🔴 Critical Issues

### C1: YAML 反序列化 RCE 风险
**位置**: `FrontmatterParser.ts:24,52,66,79` | `ApprovedRefReader.ts:71` | `configLoader.ts:60`

**问题**: `yaml.load()` 默认执行任意 JS 对象。攻击者如果能控制文档 frontmatter，可通过 `!!js/function` 等标签执行任意代码。

**修复方案**:
```typescript
import * as yaml from 'js-yaml';
// 替换所有 yaml.load() 为：
const parsed = yaml.load(rawYaml, { schema: yaml.JSON_SCHEMA });
// 或：const parsed = yaml.safeLoad(rawYaml);
```

### C2: 并发写文件竞态条件
**位置**: `ApprovedRefReader.ts:78-99` | `FrontmatterParser.ts:66-84`

**问题**: `read → modify → write` 之间无锁。多个 approve 同时执行时，后写的进程会覆盖先写进程的修改（丢更新）。

**修复方案**: 原子写入（写临时文件 → rename）
```typescript
private async atomicWrite(path: string, content: string): Promise<void> {
    const tmp = path + '.tmp.' + process.pid;
    await fs.writeFile(tmp, content, 'utf-8');
    await fs.rename(tmp, path);  // POSIX 原子操作
}
```

### C5: Regex DoS（用户输入进正则未转义）

**修复方案**:

