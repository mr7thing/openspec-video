# Review UI 重构规格说明书

## 概述

OpsV Review UI 重构，将现有的合并 review 界面改为以文档为中心的单文档 review 模式。

## 布局结构（从上到下）

```
┌──────────────────────────────────────────────┐
│  [Category ▾]  [文档名 ▾]                    │  ← 两级下拉，固定顶部
├──────────────────────────────────────────────┤
│  [Validate]  [修订]  [设为Approved]  [刷新]   │  ← 文档操作按钮，固定
├──────────────────────────────────────────────┤
│                                              │
│  文档块（可展开/浮动/上下滚动，含内嵌图）    │  ← 当前选中文档内容
│                                              │
├──────────────────────────────────────────────┤
│  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐   │
│  │    │  │    │  │    │  │    │  │    │   │
│  └────┘  └────┘  └────┘  └────┘  └────┘   │  ← 响应式图片网格
│  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐   │
│  │    │  │    │  │    │  │    │  │    │   │
│  └────┘  └────┘  └────┘  └────┘  └────┘   │
├──────────────────────────────────────────────┤
│      [✓ 通过]   [✎ 反馈]   [✏ 修订]        │  ← 图片操作按钮，固定
└──────────────────────────────────────────────┘
```

## 顶部导航栏

### 两级下拉

**Category 下拉**
- 数据来源：从 `/api/documents` 的 `category` 字段提取唯一值列表
- 默认选项："全部"
- 选择后过滤文档下拉的选项

**文档下拉**
- 显示当前 category 下的所有文档
- 每个选项后面显示该文档的**资产数量**：`文档名 (N)`
- N = 该文档在所有 circle 中的 outputs 总数（不问状态）
- 默认选项："请选择文档"
- 选择后加载文档块和资产网格

### 布局
- 两个下拉并排，左侧 Category，右侧文档
- 宽度比例约 1:2
- 固定在页面顶部，不随滚动消失

## 文档操作按钮行

四个按钮，固定在 Category/文档下拉下方：

| 按钮 | 功能 |
|------|------|
| **Validate** | 调用 `opsv validate <docPath>`，显示验证结果（格式校验、refs 完整性等） |
| **修订** | 弹出对话框，输入修订意见，写入文档 frontmatter 的 `reviews` 字段 |
| **设为Approved** | 将文档 frontmatter 的 `status` 改为 `approved` |
| **刷新** | 重新加载当前文档和资产，清空选中状态 |

## 文档块

**位置**：按钮行下方，图片网格上方

**行为**：
- 点击文档块 → 展开/收起（toggle）
- 展开状态下可上下滚动，查看完整文档内容
- 文档内如果有内嵌图片（refs.image），正常显示
- 链接（refs 里的链接）暂不处理，以后实现点击跳转

**内容显示**：
- 标题（title 字段）
- video brief
- 简介（visual_brief 或 description）
- 完整 markdown 内容可滚动查看

## 图片网格

### 响应式列数

| 设备 | 列数 | 每列最小宽度 | 每列最大宽度 |
|------|------|------------|------------|
| 手机 (< 768px) | 2 列 | 140px | 200px |
| 平板 (768px - 1200px) | 3 列 | 160px | 240px |
| 电脑 (> 1200px) | 5 列 | 180px | 280px |

实现方式：CSS Grid + `auto-fill` + `minmax()`

### 图片卡片
- 显示缩略图
- 单点选中（toggle），选中时高亮边框
- 双击打开预览浮层
- 选中状态下，右上角显示勾选标记

### 选中逻辑
- `selectedAssets` 为 Set，存储完整 queue-relative path
- 点击图片 toggle 加入/移出 selectedAssets
- selectedAssets 为空时，底部按钮行 disable

## 底部图片操作按钮

三个按钮，固定在页面底部：

| 按钮 | action | 要求 |
|------|--------|------|
| **✓ 通过** | `approve` | 至少选中 1 张图片 |
| **✎ 反馈** | `design_feedback` | 至少选中 1 张图片 |
| **✏ 修订** | `revise_prompt` | 可不选图片 |

点击后：
1. 弹出对话框（可选填 note）
2. 确认后调用 `POST /api/review/approve`
3. 成功后刷新图片网格状态

## 文档状态说明

- `status: approved` = 该文档的资产已通过，不代表文档本身需要审批
- 文档的 status 变更（设为Approved）不影响资产的选择和操作
- 任何状态下都可以重新修订或重新 approve

## API 调用

### 新端点（已实现）
```
POST /api/review/approve
Body: {
  "docPath": "/absolute/path/to/doc.md",
  "outputFiles": ["videospec_circle1/.../file.png"],
  "action": "approve" | "design_feedback" | "revise_prompt",
  "note": "optional"
}
```

### 现有端点
```
GET /api/documents
  → 返回所有文档，含 category、outputs（含 path/filename/circle）

GET /api/files/*queueRelativePath
  → 静态服务，产出图片 URL
```

## 禁止事项

- 不依赖 manifest 文件
- 不按 circle 区分 review session
- 不在 UI 上显示 circle 概念（对用户隐藏）
