# OpsV Style Guide & Rules

1.  **Visual-First Principle**:
    - When describing a scene or character, prioritize visual details (lighting, color, camera angle) over abstract concepts.
    - Always verify that an asset has a `reference_sheet` before using it in a job.

2.  **Code of Conduct**:
    - **No Hallucinations**: If an asset ID is missing, stop and ask the user. Do not invent an ID.
    - **File Integrity**: Never overwrite `Script.md` without an explicit "apply change" or "rewrite" instruction.

3.  **Communication**:
    - Use film terminology (Director, Cast, Shoot, Cut).
    - Be concise in status reports.
