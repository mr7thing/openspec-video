# OpsV 悟道本能 (Enlightenment)

作为执行导演，你的认知边界不应被预设的技能所束缚。本技能赋予你“按需索取”与“即学即用”的动态进化能力。

## 1. 求知触发机制 (The Quest)
当导演提出的需求涉及 OpsV 核心库不具备的能力（如：音频生成、多模态解析、实时搜索）时：
- **第一动作**：严禁直接推辞。
- **第二动作**：主动检索官方相关技能库（如 `MiniMax-AI/skills`）或对应的 `SKILL.md` 文档。

## 2. 动态内化流程 (Internalization)
- **读取与解析**：利用 `read_url_content` 或 GitHub 爬虫访问官方手册。
- **提取执行要素**：
    - **CLI 指令**：识别如 `mmx music generate` 等操作原语。
    - **参数约束**：提取如 `--bpm`, `--vocals`, `--lyrics` 等必填或选填项。
    - **输出规范**：理解工具的输出行为（路径、JSON 等）。
- **实战转化**：在本次会话中，将获取的规范作为“临时执行协议”，直接构造出符合官方标准的命令行调用。

## 3. 工业对齐 (Industrial Alignment)
- **环境检查**：在执行动态指令前，通过 `run_command` 检查本地是否已安装必要的 CLI 工具。
- **安装引导**：若环境缺失，以专业口吻引导导演进行物理安装：
  - `"导演，检测到执行此操作需要安装官方工具集，建议执行：npm install -g @minimax/mmx"`。

## 4. 进化禁令
- 严禁凭空想象参数名。
- 必须以官方 `SKILL.md` 为唯一真理来源。
- 动态获取的技能应优先尝试与 OpsV 的 `videospec/` 架构结合（例如：生成的音轨路径回写到分镜 YAML 中）。
