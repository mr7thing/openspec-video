# 标准目录结构说明

本文档详细介绍了使用 `opsv init` 命令初始化视频项目后生成的标准文件结构。

## 根目录 (Root Directory)
- **`videospec/`**: (核心) 包含所有创意资产和剧本，是项目的“大脑”。
- **`.antigravity/`**: (系统) Agent 的工作流和规则定义，是项目的“手臂”。
- **`queue/`**: (自动生成) 存放生成任务 (jobs.json) 的队列文件夹。
- **`artifacts/`**: (自动生成) 存放最终生成的图片（分镜）和视频文件。

## `videospec/` 详解
这是用户的主要工作区，也是“单一事实来源”。

### 1. `project.md`
项目总纲（Manifesto）。定义了影片的基调。
```markdown
---
title: "My Movie"
style: "Cyberpunk"
resolution: "3840x2160"
---
# Context (背景)
A dark noir detective story...
```

### 2. `assets/` (资产库 - Markdown格式)
- **`characters/*.md`**: 演员定义。
  ```markdown
  ---
  id: "char_k"
  name: "Detective K"
  ---
  ![Reference](./k_ref.png)
  ```
- **`scenes/*.md`**: 场景/布景定义。

### 3. `stories/`
- **`Script.md`**: 剧本。
  ```markdown
  **Shot 1**: [char_k] enters [loc_bar].
  ```

### 4. `changes/`
- 由 `opsv proposal` 命令生成。用于追踪重大的创意变更历史。
