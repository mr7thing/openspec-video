{
  "data": {
    "sessionId": "...",
    "sessionToken": "...",

    "tunnelUrl": "..."
  }

return {

  reviewUrl: data.reviewUrl,
  jwt: data.jwt || data.reviewJwt,  // 兼容两种字段名

## 6. 端到端流程推演

基于代码静态分析，推演一次完整的 `--cloud` 审阅流程：

```
$ opsv review --circle --cloud

1. CLI 解析 --cloud, 从 OPSV_CLOUD_URL / OPSV_CLOUD_API_KEY 读取配置
2. 启动本地 ReviewServer 于 :3100
3. CloudClient.createSession()
   → POST {cloudUrl}/api/sessions
   ← { data: { sessionId, sessionToken, reviewJwt, reviewUrl, tunnelUrl } }
   → 解包为 { sessionId, jwt, sessionToken, reviewUrl, tunnelUrl }
4. TunnelClient.connect()
   → 连接 wss://{cloud}/tunnel?token={sessionToken}
   ← WebSocket OPEN
   → 打印 "Tunnel connected ✓"
5. 打印 Review URL 和 Session ID
   → 审阅者打开 https://review.opsv.cloud/s/{sid}?t={jwt}
6. 审阅者请求 /s/{sid}/api/circles
   → Gateway 验证 JWT → proxyRequest → TunnelServer 发送 HTTP_REQ 帧
   → TunnelClient 收到帧 → deserializeRequest → 转发到 localhost:3100/api/circles
   → ReviewServer 响应 JSON → TunnelClient 发送 HTTP_RES 帧（分片）
   → TunnelServer 重组流 → 返回给审阅者
7. 审阅者执行 Approve
   → POST /s/{sid}/api/approve/... → 同上链路 → 本地文件被修改
8. Ctrl+C / TTL 超时 / SIGTERM
   → shutdown() → server.close() → cleanupCloud()
   → tunnelClient.close() → cloudClient.closeSession(sessionId)
   → 打印 "Cloud session closed: {sid}"
```

**推演结论**: 端到端链路在代码层面完全闭合，无逻辑断点。

## 7. 已知限制与后续工作

本次 MVP 为 **内部 Beta 级别**，以下限制不影响核心功能，但需在正式生产前解决：

| 限制 | 影响 | 后续计划 |
|------|------|----------|
| **opsv-cloud 无持久化层** | 单实例部署，重启丢数据 | P4 接入 PostgreSQL + Redis |
| **opsv-cloud 密码明文存储** | 安全合规风险 | 0.10.4 引入 bcrypt |
| **opsv-cloud 编译依赖缺失** | `stripe`/`helmet`/`express-rate-limit` 未安装 | 执行 `npm install` 即可修复 |
| **opsv-cloud 零测试覆盖** | 服务端无自动化测试 | 0.10.4 补全 |
| **无 `--status` / `--refresh` / `--close` 子命令** | 仅支持启动时自动创建/关闭 | 0.10.3 添加生命周期命令 |
| **TunnelClient 无单元测试** | 帧协议行为仅通过代码审查验证 | 0.10.4 补全 PING/PONG/ malformed frame 测试 |
| **JWT 仍在 URL 中** | 可能留在浏览器历史/代理日志 | 后续评估一次性 token 机制 |
| **无多实例保证** | 内存存储无法水平扩展 | 接入 Redis 后解决 |
