# Shortlist

> **阶段**: S3 · Shortlist
> **输入**: Script.md（S2 产出，含 Beat 列表 + 台词） + 剧本原文（叙事上下文）
> **产出**: `videospec/shortlist.md`
> **技能数**: 1（本技能独占 S3）
> **验收**: `opsv validate --dir videospec --category shortlist`

---

## 0. 核心概念：Beat → Shot

S2（beat-script）产出的 Script.md 里是 **Beat**（节拍）——画面变化的原子单位，每个画面变化一个 Beat。

本技能的职责是把这些细粒度 Beat **合并为 Shot（镜头）**——制片可执行的生产单元。

| 层级 | 概念 | 粒度 | 来源 |
|------|------|------|------|
| **Beat**（节拍） | 画面变化的原子单位 | 2-8 秒，每个画面变化一个 | S2 → Script.md |
| **Shot**（镜头） | 制片生产单元 | 6-15 秒，N 个 Beat 合并 | S3（本技能）→ shortlist.md |

**你的唯一任务**：读 Script.md 的 Beat 列表（含台词全文），参照剧本判断叙事节拍边界，将同一节拍内的 2-5 个 Beat 合并为一个 Shot，为每个 Shot 做完整的制片决策——镜头语言、台词时间轴、音效、prompt、资产清单。

## 1. 职责边界与产出

**五件产出**：

| # | 产出 | 粒度 | 说明 |
|---|------|------|------|
| 1 | **Shot 镜头语言** | 每 Shot | 景别/运镜/机位/光影 + 英文 prompt（≥50 字符） |
| 2 | **台词时间轴** | 每句台词 | 谁说的哪句话，从 Shot 内的第几秒开始，持续几秒 |
| 3 | **音效清单** | 每 Shot | 环境音、动作音、特殊音效 |
| 4 | **资产需求清单** | 整集 | 角色（含复用频次）、场景、道具 |
| 5 | **Shot 场记表** | 整集 | Shot 编号/Beat 归属/时长/场景/角色/一句话概括 |

**你做**：
- 读取 S2 产出的 Script.md，获取所有 Beat 条目（含角色、场景、台词全文、自然时长）
- 参照剧本原文判断叙事节拍边界，将 2-5 个相邻 Beat 合并为一个 Shot（6-15s）
- 为每个 Shot 决定镜头语言（景别/运镜/机位/光影/prompt）
- 根据 Beat 中的台词内容，计算每句台词在 Shot 内的起止时间
- 根据 Shot 场景和动作设计音效
- 输出资产需求清单：本集需要哪些角色、场景、道具

**你不做**：
- 修改 Beat 内容（那是 S2 的事）
- 设计角色外观（那是 S4 的事）
- 生成资产（那是 S5 的事）
- 维护 refs 依赖关系（那是 S4 的事——S4 根据你的清单建立定档文档和引用）

### Step 1: Beat 分组（叙事连续性）

Script.md 产出的 Beat 是**细粒度的独立画面单元**——每个画面变化就是一个 Beat，自然时长可能只有 2-5 秒。核心任务是将这些 Beat **按叙事节拍合并为 Shot**（每个 Shot = 一段 6-15 秒的连续画面）。

#### 影视连续性原则

| 原则 | 含义 | 分组应用 |
|------|------|---------|
| **主镜头 + 覆盖** | 先用 wide 建立空间，再用 medium/close-up 覆盖局部 | 同一空间的 wide → medium → close-up Beat 合并 |
| **正反打** | 两人对话，A 切 A，B 切 B，空间关系和视线方向一致 | 同一场景对话正反打（2-4 个 Beat）合并 |
| **180° 轴线规则** | 摄影机始终在角色同一侧 | 连续对话不跳轴（除非有明确的越轴动机） |
| **动作匹配** | 一个动作跨两个 Beat，第二 Beat 从动作中间点接上 | 连续动作拆分的 Beat 必须合并 |
| **视线匹配** | 角色看画面外某物 → 下一 Beat 是该物 | POV 组（看→所见→反应）合并 |

#### 一句话测试

**能否用一句话概括这组（2-5 个）Beat？** 能 → 同一叙事节拍，应合并为一个 Shot。

```
✅ 合并为 Shot：
  Beat 2 (3s): 宗庙庭院，禁军阵列
  Beat 3 (4s): 陆然走上前
  Beat 4 (3s): 陆然凝视婚碑
  Beat 5 (3s): 婚碑龙纹特写
  → "陆然在禁军注视下走向婚碑凝视。" → 合并为 Shot（13s）

❌ 不合并：
  Beat 2 (4s): 宗庙庭院
  Beat 3 (3s): 云璃手中珠帘
  Beat 4 (5s): 东海之滨
  → 场景从宗庙跳到东海，无法一句话概括 → 拆为两个 Shot
```

#### 节拍边界

以下标志着当前节拍结束，**不跨边界合并**：

- 场景切换（换地点）
- 时间跳跃（片刻后/须臾/夜色渐深）
- 叙事层切换（光幕内/外、梦境/现实）
- 主要角色离场/新主要角色登场
- 情节段落转换（悬念揭晓、冲突升级）

#### 正反打对话

两人对话来回切 4-6 个 Beat，场景不变、同一条信息线推进 → 同一个节拍，合并为一个 Shot。

```
✅ 正确：
  [Shot 1：云璃质问，14s]
  Beat 2-4: 云璃问(5s) → 陆然答(4s) → 云璃追问(5s)

  [Shot 2：情绪突变，10s]
  Beat 5-6: 云璃瞳孔骤缩(5s) → 珠帘落地(5s)
  → 新情绪 = 新节拍 = 新 Shot，不跨节拍合并
```

#### 时长约束

**每个 Shot（合并后）在 6-15 秒之间。**

| 合并后总时长 | 处理 |
|------------|------|
| **< 6s** | 继续合并相邻同节拍 Beat |
| **6-15s** | ✅ 合格 |
| **> 15s** | 在节拍内找自然呼吸点拆分 |

### Step 2: Shot 镜头语言决策

Beat 合并为 Shot 后，为每个 Shot 决定摄影参数。**这是 S3 的核心职责**——S2 只切 Beat（叙事层），S3 负责所有镜头语言决策。

参考 `references/shot_language_guide.md` 速查景别/运镜/机位分类。

#### 2.1 景别判断（shot_type）

根据 Shot 的叙事目的决定：

| Shot 叙事目的 | 推荐景别 | 示例 |
|------------|---------|------|
| 首次展示场景全貌、建立空间关系 | **wide** | 禁军列阵，婚碑矗立 |
| 多人动作/打斗中景、两人对话 | **medium** | 陆然袖袍一拂，禁军扑来 |
| 角色表情反应、情感变化、单人说词 | **close-up** | 云璃瞳孔骤缩 |
| 手部动作、眼睛、器物细节、字幕 | **extreme-close-up** | 珠帘落地、龙纹特写 |
| 角色主观视角 | **POV** | 陆然看向光幕 |
| 两人对话正反打 | **over-shoulder** | 云璃过肩看向陆然 |

**判断优先级**：先看有没有 POV → 有没有极端细节 → 有没有表情/情感 → 有没有空间关系建立 → 默认 medium。

**景别组合**：Shot 内可能有景别变化（推拉），写 `wide → close-up`。

#### 2.2 运镜判断（camera_movement）

| 动作特征 | 推荐运镜 |
|---------|---------|
| 角色静止、环境描写、固定对白 | **static** |
| 横扫场景、跟随视线平移 | **pan** |
| 垂直升降、从地面拉起或从上降下 | **crane** |
| 推近或拉远 | **dolly** |
| 手持晃动、紧张追逐打斗 | **handheld** |
| 跟拍角色移动 | **tracking** |

**默认值**：`static`。只有明确的动作爆发或空间切换才用动态运镜。

#### 2.3 机位判断（camera_angle）

| 画面特征 | 机位 | 说明 |
|---------|------|------|
| 普通对话、中性描写 | **eye-level** | 默认值 |
| 角色显弱小、被压制 | **high-angle** | 幼年云璃蜷缩沙滩 |
| 角色显强势、威严 | **low-angle** | 玄焰熔炉凌空降临 |
| 全景俯视 | **bird's-eye** | 皇城上空俯瞰 |
| 两人对话过肩 | **over-shoulder** | 正反打 |

**默认值**：`eye-level`。

#### 2.4 光影描述（lighting）

根据剧本原文的时段标注和光源描述：

- `日/外` → "白日室外，阳光照射..."
- `炉内火光熊熊` → "熔炉火光映照，暖橙色调"
- `深夜` → "昏暗室内，月光从窗缝透入，冷蓝调"
- `光幕` → "光幕冷光映照，高反差"

#### 2.5 prompt 生成

为每个 Shot 写英文 prompt，≥50 字符。必须包含：

1. **景别**：WIDE SHOT / MEDIUM SHOT / CLOSE-UP / EXTREME CLOSE-UP / POV SHOT / OVER-SHOULDER SHOT
2. **机位**：eye-level / low-angle / high-angle
3. **光影**：soft morning light / dramatic backlight / warm fire glow / cold moonlight
4. **角色位置与姿态**
5. **关键动作或状态**

**示例**：
```
WIDE SHOT, low-angle, dolly in. Sunlight streams through temple courtyard,
illuminating a towering wedding stele with dragon reliefs glowing gold.
LuRan enters from the courtyard gate, red robes billowing.
Cinematic composition, 3D render, Unreal Engine quality.
```

### Step 3: 台词时间轴

**这是 S3 的工作量核心。** Script.md 中的每个 Beat 包含台词全文。现在需要：

1. 将合并后的 Shot 内所有台词逐句列出
2. 按中文语速 3 字/秒计算每句台词的持续时长
3. 标注每句台词在 Shot 内的起止秒数

#### 3.1 计算规则

| 项目 | 规则 |
|------|------|
| **语速** | 中文 3 字/秒 |
| **句间停顿** | 正常换句 +1s，情绪转折 +2s |
| **动作间隙** | 纯动作 Beat（无台词）→ 按 Beat 自然时长，不计入台词时间轴 |
| **重叠对话** | 两人同时说话 → 分别标注起止时间，允许重叠 |
| **画外音** | 标注 `(OS)` off-screen |

#### 3.2 时间轴格式

每个 Shot 内逐句标注：

```
Shot 1 时间轴（总长 15s）:

| 时间 | 角色 | 台词 |
|------|------|------|
| 0:00-0:05 | — | （纯画面：禁军列阵，陆然入场） |
| 0:05-0:09 | 陆然 | "臣，陆然，受命于天。"（12字 ÷ 3 = 4s） |
| 0:09-0:15 | — | （陆然走向婚碑，凝视，6s 纯动作） |
```

#### 3.3 多角色时间轴示例

```
Shot 2 时间轴（总长 14s）:

| 时间 | 角色 | 台词 |
|------|------|------|
| 0:00-0:03 | — | （云璃从暗处走出） |
| 0:03-0:07 | 云璃 | "陆然，你当真以为，这婚约是恩赐？"（15字 ÷ 3 = 5s，含 1s 停顿取 4s） |
| 0:07-0:11 | 陆然 | "臣不敢。"（3字 ÷ 3 = 1s，续上即起，实际 1s） |
| 0:07-0:14 | 云璃 | "不敢？我看你敢得很。"（8字 ÷ 3 ≈ 3s，紧接陆然话音） |
```

> **注意**：云璃第二句从 0:07 开始，与陆然台词开头重叠——对话节奏紧凑，这是常见处理。

### Step 4: 音效设计

为每个 Shot 标注音效需求，分为环境音和动作音。

#### 4.1 音效分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **环境音** | 场景底噪，贯穿整个 Shot | 风声、雨声、市集嘈杂、宫殿回响 |
| **动作音** | 特定时刻的 Foley | 脚步声、兵器碰撞、珠帘落地、衣袍拂动 |
| **特殊音效** | 魔法/科技/超自然 | 光幕嗡鸣、灵力爆发、时空撕裂 |

#### 4.2 音效清单格式

```
Shot 1 音效:

| 类型 | 时间 | 音效 | 说明 |
|------|------|------|------|
| 环境 | 全程 | 风啸声 | 宗庙高地，风声低沉 |
| 环境 | 全程 | 铠甲摩擦 | 禁军阵列，金属轻微碰撞 |
| 动作 | 0:02 | 重步落地 | 陆然踏步入场 |
| 动作 | 0:05 | 衣袍拂动 | 陆然红袍转身 |
```

#### 4.3 默认环境音

每个场景自动匹配：

| 场景时段 | 默认环境音 |
|---------|-----------|
| 日/外 | 微风、鸟鸣（可选） |
| 日/内 | 室内回响 |
| 夜/外 | 虫鸣、夜风 |
| 夜/内 | 烛火噼啪、寂静 |
| 雨 | 雨声（贯穿） |
| 战斗 | 金属碰撞、呼喝声 |

### Step 6: 生成 shortlist.md

按 `references/shortlist_template.md` 结构生成。

> **核心原则**: Frontmatter 是下游的确定性合同。`shots`/`characters`/`scenes`/`props` 四个数组是 S4/S5/S6/S7 的输入——下游只从 frontmatter 拿结构化数据就能开工，不需要解析正文表格。正文表格是给人类看的展开版。

必须包含：

1. **Frontmatter** — 下游确定性合同：
   - `shots` 数组（id/duration/scene/characters/props）→ S5 分镜 + S6 视频
   - `characters` 数组（id/frequency）→ S4 资产定档 + S7 配音
   - `scenes` 数组（id/frequency）→ S4 定档文档
   - `props` 数组（id）→ S4 资产
2. **Shot 场记表**（表格：Shot/Beats/Duration/Scene/Characters/Props/Summary）
3. **Shot 镜头语言**（逐 Shot：景别/运镜/机位/光影/prompt）
4. **台词时间轴**（逐 Shot：时间/角色/台词）
5. **音效清单**（逐 Shot：类型/时间/音效/说明）
6. **资产需求**（角色/场景/道具 + 复用频次）

## 4. 这 7 个关切在本技能如何贯彻

### ① 生产流程
六步：Beat 分组 → 镜头语言 → 台词时间轴 → 音效 → 资产清单 → 生成文档。每步依赖前一步产出。

### ② 依赖处理
- **上游**: S2 Script.md + 剧本原文
- **下游**: S4 定档设计——资产需求清单告诉 S4 需要创建哪些定档文档
- 本阶段不进 Circle

### ③ 提示词生成
每个 Shot 必须生成 prompt。按 §2.5 规则：包含景别/机位/光影/角色位置/关键动作，英文，≥50 字符。

### ④ 引用语法
- 场记表中角色/场景/道具用 `@id` 格式引用
- 资产清单中标注复用频次（高频复用 = 跨 3+ 个 Shot）

### ⑤ 任务环编排
不进 Circle。作为 S4 的调度依据。

### ⑥ 迭代与 Review
- `opsv validate --dir videospec --category shortlist`
- 检查：每个 Shot 时长是否在 6-15s？是否跨节拍错误合并？台词时间轴是否完整？音效是否遗漏？资产清单是否完整？
- 审批通过后 S4 按资产清单执行

### ⑦ 资产回写
本阶段无 `asset_id` 回写。

## 5. 注意事项

1. **分组前必须读剧本原文** — 只看 Script.md 的 Beat 条目无法判断节拍边界，需要剧本的叙事上下文
2. **beat_id 与 S2 完全一致** — 大小写、分隔符都不能差
3. **每个 Shot 必须能一句话概括** — 概括不了说明跨节拍了，拆
4. **每个 Shot 必须写镜头语言** — 景别/运镜/机位/光影/prompt，缺省值见 §2.1-2.5
5. **时长不够就继续合** — < 6s 的 Shot 必须再并相邻同节拍 Beat
6. **正反打不拆** — 同一场景、同一条对话线的正反打是一个 Shot
7. **场景切换 = 节拍断裂** — 新场景必然是新 Shot
8. **资产清单只列不管** — 列出需要哪些资产即可，依赖解析和 refs 维护是 S4 的事
9. **台词时间轴逐句计算** — 不能跳过任何一句台词，按 3 字/秒如实计算起止时间
10. **音效不可省略** — 每个 Shot 至少标注环境音，有动作必有动作音
