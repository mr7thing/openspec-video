# Seedance 动态提示词书写规范 (Seedance Dynamic Prompt Specification)

> **参考来源**：《浣溪沙·蒲月香》创作复盘 — NAGI & BODHI 端午旅行短片 12 组 Seedance 提示词实践
> **规范版本**：v1.0 — OPSV Refs 驱动模式兼容

---

## 一、概述

Seedance 动态提示词是漫剧视频生成的核心输入。写好一组提示词，等于把导演脑海中的镜头语言翻译成了模型能理解的文字指令。

本规范提炼自真实商业级 Seedance 项目实践，定义了 **镜头组分镜提示词** 的完整结构、每层书写标准和 OPSV 引用规范。

### 1.1 核心概念：镜头组 (Shot Group)

与逐镜生成不同，**镜头组** 是将 4 个连续镜头打包为一个 12 秒提示词单元的写法。

| 维度 | 逐镜提示词 | 镜头组提示词（本规范） |
|------|-----------|---------------------|
| 时长 | 3-10 秒/镜 | 12 秒/组（4 镜×~3 秒） |
| 结构 | 单镜头描述 | 多镜头顺序描述 + 全局总控 |
| 一致性 | 依赖逐镜手动对齐 | 同一组的角色造型/风格由总控词保证 |
| 使用场景 | 简单镜头/空镜 | 有叙事推进的连续场景 |
| ⚠️ 适用性 | Seedance 2.0 推荐 | 更适合 GPT Image / Midjourney 首帧生图后用 Animate 补帧，Seedance 需根据平台字数限制选择 |

### 1.2 镜头组的价值

- 统一同一组的角色造型（如"本组 NAGI 穿造型 C"只用写一次）
- 统一风格总控（光影、色调、禁止项）
- 旁白在同一组内自然连贯
- 剪辑时按组切分，节奏清晰

## 二、提示词七层结构

每一组 Seedance 镜头组提示词由 7 个层次构成，**必须按以下顺序书写**：

```
┌─────────────────────────────────────────────────┐
│ 第1层: 使用参考 (Reference Usage)                 │
│   如何利用故事板 + 角色参考图                     │
├─────────────────────────────────────────────────┤
│ 第2层: 本组角色造型 (Character Outfits)           │
│   每个角色的服装/外观/状态定义                     │
├─────────────────────────────────────────────────┤
│ 第3层: 本组总控词 (Master Control Prompt)         │
│   全局光影、色调、禁止项、参考图                   │
├─────────────────────────────────────────────────┤
│ 第4层: 旁白音色要求 (Narration Voice)             │
│   音色、语速、语气变化、特殊标注                   │
├─────────────────────────────────────────────────┤
│ 第5层: 镜头 1 规格 (Shot 1)                       │
│   画面/运镜/表演/环境音/旁白                      │
├─────────────────────────────────────────────────┤
│ 第6层: 镜头 2 规格 (Shot 2)                       │
├─────────────────────────────────────────────────┤
│ 第7层: 镜头 3 规格 (Shot 3)                       │
├─────────────────────────────────────────────────┤
│ 第8层: 镜头 4 规格 (Shot 4)                       │
└─────────────────────────────────────────────────┘
```

## 三、各层书写规范

### 第1层：使用参考 (Reference Usage)

**目的**：告诉模型如何使用参考图——不是"有就行"，而是"按什么顺序、怎么用"。

```markdown
【标准模板】
使用提供的故事板作为顺序参考，严格按照本镜头组的四格分镜顺序生成。
使用已上传的 @<角色ID>、@<角色ID> 角色设定图作为角色参考，
保持 <角色A> 脸型、五官、发型、气质一致，
保持 <角色B> 犬种/体型/毛发/配饰一致。
不要照搬角色设定卡的平面构图，要将角色元素拆解为真实电影镜头。
```

**OPSV @-refs 映射**：

| 句子中的引用 | OPSV `refs.image` 中的对应 |
|-------------|--------------------------|
| 故事板参考图 | `"@:storyboard_grid": [storyboard/ep01_grid.png]` |
| 角色 A 设定图 | `"@<char_id>": [elements/characters/<char_id>_ref.png]` |
| 角色 B 设定图 | `"@<char_id>": [elements/characters/<char_id>_ref.png]` |
| 场景氛围参考 | `"@<scene_id>": [elements/scenes/<scene_id>.png]` |

---

### 第2层：本组角色造型 (Character Outfits)

**核心规则**：每个角色在本组的造型必须**明确写出**——模型不记得上一组的衣服。

```markdown
【标准模板】
本组角色造型
<角色A>穿造型<字母>。<服装分项描述，至少 3 项组成>。
气质<关键词 1>、<关键词 2>、<关键词 3>。
<角色B>穿造型<字母>。<犬种/体型/装饰描述>。
状态<关键词 1>、<关键词 2>、<关键词 3>。
```

**分项描述结构**（每项 = 颜色 + 材质/款式 + 部件名）：

```markdown
❌ 错误："NAGI 穿白色上衣和青色裙子"
✅ 正确："月白立领新中式短衫，远山紫轻薄外搭，青黛长裙，竹青细发夹"
```

**造型变化规则**：

| 规则 | 说明 |
|------|------|
| 同一组内所有镜头使用相同造型 | 不允许在组内换装 |
| 换组才能换造型 | 造型变更标注在 shotlist 中 |
| 多组连续使用相同造型 | 每组仍需写（防模型遗忘），但可简写为"同上一组" |
| 宠物/动物角色 | 同样需要分项描述——项圈、挂饰、胸背、状态变化 |

**OPSV @-refs 映射**：

角色造型信息不直接对应 `refs.image`，而是通过 `act_characters` 和 `refs.image` 中的 override 图来锚定：

```yaml
act_characters:
  - "@nagi"          # 编译器从 frontmatter 读取 asset_id
  - "@bodhi"
refs:
  image:
    "@nagi":         # 如果有造型 override 图
      - elements/characters/nagi_outfit_c.png
    "@bodhi":
      - elements/characters/bodhi_state_3.png
```

### 第3层：本组总控词 (Master Control Prompt)

**核心**：一句话定义本组的视觉宇宙——空间、时间、光影、色调。

```markdown
【标准模板】
<空间地点>，<时间/季节>，<核心风格>，<画幅比例>，
重点表现<关键视觉元素：至少 3 项>。
禁止<负面列表：至少 4 项>。
无背景音乐，仅保留<环境音指示>。
```

**总控词词汇库**（按维度整理）：

#### 空间时间
```
上海公寓夜晚，姑苏小院初夏白昼，高铁车厢，
苏州园林午后，荷塘入夜，都市黄昏，古镇清晨
```

#### 画幅与质感
```
16:9 电影画幅，当代写实电影质感，低饱和中国传统色，
低饱和冷灰蓝与暖白光影，空气湿润感，自然光
```

#### 色彩体系
```
月白，艾绿，藕荷，竹青，青黛，天水碧，松花，
柳黄，黛青，远山紫，茶褐，朱砂，墨黑，金
```

#### 禁止项（标准黑名单）
```
禁止字幕，禁止文字叠加，禁止UI，禁止水印，
禁止卡通风，禁止过度美颜，禁止夸张表演，
禁止满月，禁止梦幻特效
```

### 第4层：旁白音色要求 (Narration Voice)

```markdown
【标准模板】
旁白音色要求
旁白全部使用 <角色名> 本声，<性别年龄>，<音色质感>，<语速>，
整体以<地域/文化气质>来念白，声音<形容词>，语气<形容词>。
不要播音腔，不要广告腔，不要科普腔。
<本组特殊语气要求>。
```

**旁白写作三原则**：

| 原则 | 说明 |
|------|------|
| 短句为主 | 每句 8-16 字，不解释画面 |
| 情绪留白 | 不要写满——观众需要"读画面"的空间 |
| 语法断裂 | 非完整句优于完整句（"有些名字，本身就是方向。" > "有些名字本身就是一个方向。"） |

**旁白与画面的关系**：

```
画面负责"看到什么"——旁白负责"感受到什么"
旁白不重复画面，画面不解释旁白

✅ 正确：画面=艾叶落入瓷碗，旁白="艾叶清苦，像慢板"
❌ 错误：画面=艾叶落入瓷碗，旁白="她轻轻将艾叶放入碗中"
```

### 第5-8层：单镜头规格 (Shot Specification)

每个镜头由 **5 个维度** 组成，缺一不可：

```
【标准模板 — 每个镜头一个】
镜头<组号>-<镜号>｜<镜头标题>
时长 <N>秒
画面<画面描述——空间定位、主体位置、镜头中包含什么>
运镜<起幅景别 + 运镜方式 + 落幅>
表演<角色动作、表情变化、身体语言>
环境音<本镜的主要环境声音>
旁白<本镜对应的旁白文本——如果没有写"无">
```

#### 5.1 画面 (Scene Description)

| 要素 | 必须包含 | 示例 |
|------|---------|------|
| 空间定位 | ✅ | "上海公寓书桌旁" "姑苏小院竹窗边" |
| 主体位置 | ✅ | "NAGI 靠窗而坐" "BODHI 趴在木案下" |
| 关键元素 | ✅ | "散落的艾叶、丝线、香囊布料" |
| 光影状态 | ✅ | "电脑屏幕冷光映亮侧脸" "夕阳光从窗外斜入" |
| 画面信息量 | 3-5 个视觉要素 | 太多则失焦，太少则空洞 |

**画面描述的长度控制**：

```
氛围镜头：  3-4 个视觉要素（如"窗边香囊，窗外灯火，竹帘轻动"）
对话镜头：  3-4 个视觉要素 + 角色位置关系
动作镜头：  4-5 个视觉要素 + 动作序列
特写镜头：  2-3 个极细致要素（如"针尖穿过布面，丝线紧绷"）
```

#### 5.2 运镜 (Camera Movement)

```
【语法】
<起幅景别> <运镜方式> → <落幅>
或
<固定机位> + <景别>

【常用组合】
- 35mm 中近景，缓慢推进 → 从手机震动移至电脑屏幕
- 低机位近景，对焦从 BODHI 切到 NAGI 低头的眼神
- 侧后方中景缓慢推进到半身，再收至面部近景
- 俯拍特写，缓慢推进，突出纸张纹理
- 固定中景，门框构图，门缓缓合上
- 窗边固定视角，外景流动，轻带玻璃倒影

【禁止】
- "镜头随意移动"
- "稳定器跟拍"（太笼统，不说跟谁）
- 一个镜头里出现 3 次以上运镜变化
```

#### 5.3 表演 (Performance)

**真人镜头**——不是 AI 图形的角色指示，是导演对演员的表演指示：

```markdown
✅ 正确："眼神从疲惫变为微微清明，肩膀略微下沉，像做了一个很小的决定"
✅ 正确："手指停在鼠标旁，呼吸变慢，目光放空，但未从屏幕移开"
❌ 错误："她看起来很开心"
❌ 错误："角色 AI 生成中"（等于没写）
```

**动物/宠物表演**——遵循真实生物逻辑，不拟人化：

```markdown
✅ 正确："BODHI 从桌下慢慢走来，靠在 NAGI 腿边，抬头看她"
✅ 正确："它轻嗅簸箕中药草，鼻尖轻颤，随后卧倒在她脚边"
❌ 错误："BODHI 理解了她的情绪，用眼神安慰她"
❌ 错误："小狗露出心疼的表情"
```

#### 5.4 环境音 (Ambience Sound)

不是"可选的背景描述"，是决定沉浸感的关键层：

```markdown
【标准写法】
<主环境音>，<辅助层次音>，<点缀细节音>

✅ 正确："手机震动，极轻提示音，空调低频，远处城市车流"
✅ 正确："木门轻响，蝉鸣初起，风过竹叶"
✅ 正确："布料摩擦，纸页翻动，BODHI 轻短呼吸"

【无 BGM（背景音乐）的组的写法】
"无背景音乐，仅保留真实环境音"

【有 BGM 的组的写法】
"背景音乐：钢琴为主，干净明亮，节奏轻推进感"
```

#### 5.5 旁白时序 (Narration Timing)

旁白不是"写在哪里都行"——需要精确标注：

```markdown
旁白 "城市有时候，不说话，但一直在催人。"
      ↑ 这句念在镜头 1-1 的后半段

旁白 "我只是……有点听不清自己了。有点……听不清了。"
      ↑ 这句念在镜头 1-2 的中间，与静止眼神对齐

无
      ↑ 本镜无旁白——让环境音成为唯一声轨
```

**无旁白镜头**的写法：必须显式写"无" / "本镜无旁白"，而不是省略不写。

## 四、镜头组节奏控制

### 4.1 标准节奏：4 镜 × 12 秒

```
┌──────────┬──────────┬──────────┬──────────┐
│ 镜头 1   │ 镜头 2   │ 镜头 3   │ 镜头 4   │
│ 2.5-3.5s │ 2.5-3.5s │ 2.5-3.5s │ 2.5-3.5s │
├──────────┴──────────┴──────────┴──────────┤
│              12 秒（一组）                 │
└───────────────────────────────────────────┘
```

### 4.2 节奏变体

| 节奏类型 | 时长分配 | 适用场景 |
|---------|---------|---------|
| 标准叙事 | 3s + 3s + 3s + 3s | 对话/日常动作 |
| 起承转合 | 3.5s + 2.5s + 3s + 3s | 有情绪推进的场景 |
| 动感加速 | 2.5s + 2.5s + 3.5s + 3.5s | 动作/转场 |
| 沉稳呼吸 | 3.5s + 3s + 3s + 2.5s | 氛围建立 |
| 特写爆发 | 2s + 2s + 4s + 4s | 情绪高潮 |

### 4.3 景别递进

一组镜头内，景别应有**自然的递进逻辑**：

```markdown
建立镜头（WS/EWS）→ 交代关系（FS/MS）→ 聚焦细节（CU/ECU）
    ↓                    ↓                   ↓
"在哪"            "谁在做什么"           "感受到什么"

✅ 镜头组示例：
镜头1: EWS 荷塘全景（建立空间）
镜头2: MS 窗边 NAGI（交代主体）  
镜头3: CU 香囊流苏（聚焦细节）
镜头4: MCU NAGI 摸 BODHI（收束情绪）
```

## 五、OPSV 文档映射

### 5.1 镜头组 → OPSV 文档

每一组 Seedance 提示词不直接对应一个 OPSV 文档。正确的映射关系：

```
Seedance 镜头组（12秒，4镜）
        │
        ├──→ storyboard/shotlist.md（汇总列表）
        ├──→ storyboard/shot_NN.md（每镜独立文档）
        │     ↑ 每个 shot 文档对应镜头组中的 1 个镜头
        │
        └──→ comic_shot 文档（视频生成时）
              ↑ 实际生成时，可能将整组或单镜编译为 comic_shot
```

### 5.2 提示词中的 @-refs 详解

| 提示词中写法 | 编译器行为 | 对应 `refs` 字段 |
|-------------|-----------|-----------------|
| `@nagi` (角色名) | 读取 `@nagi.md` → `asset_id` → Seedance API | `act_characters: ["@nagi"]` |
| `@bodhi` (宠物角色) | 同上 | `act_characters: ["@bodhi"]` |
| `@:previs_grid` | 解析为 `elements/previs/EP01_grid.png` | `refs.image."@:previs_grid": [...]` |
| `@temple` (场景) | 读取场景 `visual_detailed` → 传入为文字描述 | `refs.image."@temple": [...]` |
| `@style:donghua` | 读取风格参考图 | `refs.image."@style:donghua": [...]` |
| `@nagi:face` (角色局部) | 指定面部特写参考 | `refs.image."@nagi:face": [...]` |

### 5.3 映射示例

```yaml
# storyboard/shot_01_01.md — 对应镜头组01的第1镜
---
category: comic_storyboard
title: "EP01 镜头组01-1 — 消息爆满的桌面"
duration: "2.5s"
shot_type: "MCU"
camera_motion: "35mm 中近景,缓慢推进,从手机震动移至电脑屏幕"

# 本镜在组内的角色造型（继承自组级规格）
outfit_group: "A"             # NAGI 造型A
bodhi_state: "1"              # BODHI 状态1

# 组级总控引用（在 shot 文档中通过 parent_group 引用组级规格）
parent_group: "EP01_group_01"

prompt: >
  上海公寓夜晚书桌，电脑屏幕冷光亮起，多个设计窗口、聊天消息、邮件提醒同时打开。
  手机在桌边震动，半杯冰美式水珠缓缓滑落，桌面有散乱草稿、手账、鼠标和笔。
  人物暂不露脸，只用桌面压力建立情绪。

  环境音：手机震动，极轻提示音，空调低频，远处城市车流。
  旁白：城市有时候，不说话，但一直在催人。

  无背景音乐，仅保留真实环境音。
refs:
  image:
    "@nagi":
      - elements/characters/nagi_outfit_a.png
    "@:desktop_mood":
      - refs/night_desktop_mood.png
---
```

## 六、完整示例

以下是一组完整的 Seedance 镜头组提示词（基于《浣溪沙·蒲月香》模式提炼）：

```markdown
========== 镜头组04｜到达姑苏，第一眼要美 ==========

【使用参考】
使用提供的故事板作为顺序参考，严格按照镜头组04四格顺序生成：
院门推开 → 竹窗荷塘 → NAGI 换装亮相 → BODHI 听声。
使用已上传的 @nagi、@bodhi 角色设定图作为角色参考，
保持 NAGI 脸型、五官、发型、气质统一，
保持 BODHI 犬种、毛发质感、项圈配饰统一。
不要照搬角色设定卡平面构图，要将角色元素拆解为真实电影镜头。

【本组角色造型】
NAGI 造型 C：月白对襟上衣，艾绿短褙子，藕荷长裙，月白小围裙，竹青发带。
BODHI 造型 3：艾绿色项圈，小平安结挂饰。
```
