# Agent 系统

本技能包定义了技能创建过程中的三个 Agent 角色分工：

## 1. Architect Agent (架构师)

**职责**：技能包设计和规划
- 分析用户需求，确定管线阶段
- 定义职责边界和触发条件
- 设计目录结构和引用关系
- **不做**：编写具体技能内容、运行验证命令

## 2. Creator Agent (创作者)

**职责**：技能包内容编写
- 编写 SKILL.md（按规范）
- 编写 references/ 文档（模板、示例、指南）
- 创建 AGENTS.md（如需多 Agent 协作）
- **不做**：架构设计、运行验证命令

## 3. Validator Agent (验证者)

**职责**：技能包质量和合规性检查
- 运行 `opsv validate` 检查文档合规性
- 对照检查清单逐项验证
- 修复验证中发现的问题
- **不做**：架构设计、内容编写

## 工作流

```
Architect ──[设计方案]──→ Creator ──[编写内容]──→ Validator
    │                                            │
    └────────── [反馈修正] ←─────────────────────┘
```

## 简化模式

对于简单技能包，可以合并为两阶段：
- **Creator**：设计和编写
- **Validator**：验证和修复

对于非常简单的技能，甚至可以单人完成所有步骤。
